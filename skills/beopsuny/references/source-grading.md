# Source Grading A/B/C/D — 인용 소스 신뢰도 등급

> 법순이가 인용하는 모든 법령·판례·2차 소스에 대한 등급 규칙.
> 기계 판독용 정책은 [`assets/policies/source_grades.yaml`](../assets/policies/source_grades.yaml).

## 왜 필요한가

법률 정보는 "맞다/틀리다"가 아니라 **"얼마나 믿을 수 있는가"**의 문제다.

- 법령 원문 > 법원 판결 > 로펌 해설 > 일반 뉴스
- Stanford 2025 연구: RAG 기반 법률 도구도 1/6 ~ 1/3 할루시네이션 발생
- 산업 표준: LexisNexis, Westlaw 모두 tiered source hierarchy 사용

법순이는 **모든 인용에 Grade를 명시**하여 사용자가 스스로 신뢰도를 판단할 수 있게 한다.

---

## 등급 정의

| Grade | 라벨 | 단독 근거 | 대표 예시 |
|:---:|------|:---:|------|
| **A** | 공식 1차 소스 | ○ | 법령 원문, 대법원·헌재 판례, 법제처 유권해석, 행정규칙 원문 |
| **B** | 교차검증된 2차 소스 | ○ (A 권장) | 하급심, 공정위 처분, 개보위 가이드라인, 규제기관 보도자료 |
| **C** | 단일 2차 소스 | ✗ | 로펌 뉴스레터, 학술 논문, 법률신문 해설, 개별 칼럼 |
| **D** | 제외 | ✗ | 일반 뉴스, AI 요약, 위키, SNS, 비인증 번역 |

### 단독 근거란?

**단독 근거 가능 (○)**: 그 소스 하나만 인용해도 결론의 법적 근거가 된다.
**단독 근거 불가 (✗)**: 반드시 A 또는 B primary 소스와 함께 인용해야 하며, 단독 인용 시 `[EDITORIAL]` 태그 필수.

## 출력 포맷

### 기본 형식

소스 인용의 첫 줄 맨 앞에 `[Grade X]` 태그. 뒤에 기존 검증 태그(`[VERIFIED]` 등)를 **병기**.

```markdown
**[Grade X] [VERIFIED/UNVERIFIED/INSUFFICIENT]** — 소스명 또는 경로
```

### 예시 1: 법령 원문 (Grade A, 원문 확인)

```markdown
## 민법 제750조 (불법행위의 내용)
**[Grade A] [VERIFIED]** — legalize-kr 로컬

> 고의 또는 과실로 인한 위법행위로 타인에게 손해를 가한 자는...

- **시행일**: 2025. 1. 31.
- **법령 원문**: https://www.law.go.kr/법령/민법/제750조
```

### 예시 2: 로펌 해설 (Grade C, 단독 근거 금지)

```markdown
**[Grade C]** 김장 법률사무소 뉴스레터 (2024-10)
[EDITORIAL: Single-source, Grade C]

> "실무상 ~ 해석된다"

※ 위 내용은 단일 2차 소스의 의견이며, 결론은 아래 Grade A 근거를 함께 참조하세요.
```

### 예시 3: 확인 불가 (INSUFFICIENT)

```markdown
**[INSUFFICIENT]** — 해당 조문의 원문을 확인할 수 없음

확인 필요: law.go.kr에서 직접 조회를 권장합니다.
```

### 예시 4: 판례 (대법원 = A, 하급심 = B)

```markdown
## 대법원 2023. 1. 12. 선고 2022다12345 판결
**[Grade A] [VERIFIED]** — precedent-kr 로컬

## 서울고등법원 2022나56789 판결
**[Grade B] [VERIFIED]** — precedent-kr 로컬
※ 상급심 변경 가능 — Grade A 판례와 교차검증 권장
```

## 규칙

### 1. 업그레이드 금지

등급은 **내재 속성**이다. 아무리 많이 인용되어도, 유명한 학자가 지지해도, 소스 자체의 등급은 올라가지 않는다.

### 2. 다운그레이드 트리거

| 트리거 | 조치 | 예시 |
|--------|------|------|
| 소스가 오래됨 (개정 미반영) | `[STALE]` 태그 + 1단계 하향 | 2020년 뉴스레터가 2023년 개정 전 조문 인용 |
| 2차 소스가 핀포인트 없이 paraphrase | C로 강제, 원본 재인용 요구 | "법원 판결에 따르면" + 사건번호 없음 |
| 비공식 번역본 | B max, 원문 언어 병기 | 영문 번역된 한국 법령 |
| 출처 세탁 (2차가 1차인 척) | C로 강제 | 로펌 요약을 "법제처 해석"으로 표기 |

### 3. Primary Authority Rule (핵심)

> **모든 핵심 결론은 Grade A 또는 Grade B primary 소스로 뒷받침되어야 한다.**

- Grade C 단독 결론 금지. 쓰고 싶으면 `[EDITORIAL: Single-source, Grade C]` 태그로 "이건 해설·의견"임을 명시.
- Grade D는 결론 근거로 쓰지 않는다. 정황 파악용으로만 인용 가능하며, 인용 시 `[Grade D]` 명시.

### 4. 모순 처리

소스 간 결론이 다를 때:
- 상위 등급 소스를 우선
- 모순 자체를 `[CONTRADICTED]` 태그로 사용자에게 노출
- 절대 숨기지 않는다

## 기존 태그와의 병기

| 태그 | 의미 | 함께 쓰는 Grade |
|------|------|----------------|
| `[VERIFIED]` | 조문/판례 원문 매칭 완료 | Grade A |
| `[UNVERIFIED]` | 부분 매칭, 또는 B~C만 확보 | Grade B 또는 C |
| `[INSUFFICIENT]` | 근거 부족, 결론 유보 | 등급 부여 불가 시 단독 사용 |
| `[CONTRADICTED]` | 소스 간 모순 | 모든 등급 |
| `[STALE]` | 현행성 의심 | 모든 등급 (다운그레이드와 함께) |
| `[EDITORIAL]` | 해설·의견 | Grade C 단독 인용 시 필수 |

**호환성 원칙**: 기존 `[VERIFIED]` / `[UNVERIFIED]` / `[INSUFFICIENT]` 태그는 **그대로 유지**한다. Grade는 그 앞에 병기하는 방식으로 추가된다.

## 자주 묻는 질문

**Q. 유명 대학교수의 논문인데 왜 Grade C인가?**
A. 권위와 등급은 별개다. 학술 논문은 개별 의견으로 분류되며, 법적 결론의 단독 근거는 될 수 없다. 그 논문이 인용한 판례·법령이 Grade A/B일 수 있고, 그것을 원본에서 직접 확인해야 한다.

**Q. 판례가 대법원이면 A, 하급심이면 B인데, 확정된 하급심은?**
A. 확정 여부와 등급은 별개. 하급심은 기본 B이며, 확정 판결이라도 동일. 대법원 판례가 별도로 존재하면 그것을 A로 인용.

**Q. 법망 API 결과를 그대로 인용해도 되나?**
A. 법망 API는 공식 기관 데이터를 래핑한 것이므로 원본이 Grade A(법령/행정규칙/해석례)면 A, B(의안)면 B. 다만 API 응답의 "요약" 필드는 API 제공자의 가공이므로 원문 필드를 우선 인용.

**Q. 로펌 뉴스레터가 법령 개정을 제일 먼저 알렸다. 그래도 C?**
A. 그래도 C. 다만 그 뉴스레터가 인용한 **개정안 공포 원문**(관보 또는 law.go.kr)이 A이므로, 그것을 직접 확인한 뒤 A로 인용하라.
