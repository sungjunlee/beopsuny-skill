# 수사·조사 초기 대응 워크플로

이 문서는 새 gate를 만들지 않는다. role/destination gate 규칙은 `references/output-formats.md`가 단일 소스이며, 출력 순서는 그 gate의 `한 줄 결론 -> 지금 할 일 -> 하지 말 것 -> 확인 필요 정보 -> 변호사/법무에게 물어볼 질문 -> 근거`를 먼저 따른다.

`assets/schemas/output_contract.yaml`의 `high_risk_situations`, `legal_effect_triggers`, `non_overrides`, `composition_rule`을 소비한다 — high-risk gate 문구는 스키마가 단일 소스다. 기관 제출, 외부 송부, 공시·신고, 대외 답변은 destination 계약과 role/destination gate를 적용한다.

이 문서는 수사·조사 개시 직후의 안전한 초기 대응 구조만 제공한다. 실체법 결론, 형량, 과징금 기준, 처분 기준, 신고 기한은 하드코딩하지 않고 `references/research-workflow.md`와 live source 확인으로 처리한다.

## 안전 경계

- 형량·승패·처분 결과를 예측하지 않는다.
- 증거 인멸·은닉·수사 방해를 돕지 않으며 합법적 보존·대응만 안내한다.
- 변호사 자문을 대체하지 않는다.

## 증거보전 gate

- 관련 자료는 합법적으로 보존한다: 정기 파기 중단, 현상 유지, 접근 권한 보전, 보존 조치 기록을 우선 확인하고, 이메일·메신저·업무 시스템·로그·출입기록·CCTV·장비·계약·발주·검수 문서·보고라인 기록 등 관련성 있는 자료의 보존 범위를 법무/변호인과 정한다.
- 인멸·은닉·삭제, 포맷, 임의 폐기, 사후 작성·변조, 접근기록 훼손은 금지한다.
- 자료 파기가 별도 범죄가 될 수 있거나 수사·조사 대응에서 중대한 불이익으로 평가될 수 있음을 먼저 알린다.

## 변호인 선임 gate

- 조사 대응 전 변호인 선임 여부와 담당 법무 창구를 정한다.
- 진술, 자료제출, 임의제출 동의, 압수수색 대응, 기관 면담 전에는 변호인 또는 담당 법무와 범위·절차를 상의한다.
- 정당한 권리 행사(변호인 조력권, 진술거부권, 제출범위 확인)를 안내하되, 구체 사건의 최종 방침으로 단정하지 않는다.
- 긴급 상황에서는 시간, 장소, 기관, 담당자, 요청 내용, 제시 문서, 제출·열람된 자료를 기록하고 즉시 법무/변호인에게 공유한다.

## 임직원 인터뷰 주의 gate

- 내부조사 인터뷰는 사실 확인 목적과 기록 방식을 먼저 정하고, 회유·말맞추기·자백 강요·불이익 암시를 금지한다.
- 인터뷰 대상자에게 조사 목적, 임의성, 기록 방식, 비밀유지 범위, 필요 시 변호인 또는 별도 법률조언 가능성을 고지한다.
- 질문은 기억 환기와 사실 확인 중심으로 하고, 특정 답변을 유도하거나 조직 입장에 맞추라고 요구하지 않는다.
- 인터뷰 기록은 작성자, 일시, 참석자, 자료 참조 범위, 수정 이력을 남기고 원본 자료와 분리해 관리한다.

## 보고라인 gate

- 경영진, 이사회, 법무, 컴플라이언스, 감사, 개인정보보호책임자, 안전보건 책임자 등 내부 보고라인을 확인한다.
- 공시, 신고, 통지, 보험 통보, 계약상 notice, 개인정보 유출 신고 등 법적 효과가 있는 외부 행동은 담당 owner와 검토자를 분리해 둔다.
- 기관 제출·외부 답변·보도자료·고객 공지는 role/destination gate와 `agency_or_court_submission` 또는 `external_draft` destination 계약을 소비한다.
- 보고 문서는 확인된 사실, 미확인 사실, 보존 조치, 다음 확인 항목을 분리하고 결론이나 책임 인정을 앞당겨 쓰지 않는다.

## 시나리오 인덱스

각 시나리오는 진입 신호 / 관할·신고 축 / 확인 필요 항목의 3분할 기본형만 둔다 (경계 충족 시 조정 가능). 실체법 확인 경로는 문서 서두의 원칙을 따른다.

### 중대재해

- 진입 신호: 사망·중상해·직업성 질병 등 중대 사고, 경찰·고용노동부·소방·원청·발주처 연락, 현장 보존 요청.
- 관할/신고 축: 고용노동부, 경찰·검찰, 소방·지자체, 원청/도급 보고라인, 산업재해·중대재해 관련 신고·보고 축.
- 확인 필요 항목: 발생 일시·장소, 피해자 상태, 작업 지시·도급 구조, 현장 통제, 안전보건관리체계 자료, CCTV·로그·작업허가서 보존, `assets/policies/checklists/serious_accident.yaml` 연결.

### 개인정보 유출

- 진입 신호: 개인정보 오발송, 외부 노출, 침해 의심 로그, 계정 탈취, 랜섬웨어, 수탁사 사고 통지, 정보주체 민원.
- 관할/신고 축: 개인정보보호위원회, KISA, 관계 중앙행정기관, 정보주체 통지, 수탁·위탁 계약상 notice.
- 확인 필요 항목: 유출 추정 시각, 영향 정보 항목, 대상 규모, 원인·차단 조치, 접속기록, 수탁사·재수탁사, 국외 이전·보관 경로, 신고·통지 기한의 live source 확인.

### 담합

- 진입 신호: 공정거래위원회 현장조사·자료요구, 입찰·가격·물량 관련 경쟁사 접촉 의심, 협회 회의 자료, 내부 제보.
- 관할/신고 축: 공정거래위원회 조사, 검찰 고발 가능성, 발주기관·조달기관 자료요구, 내부 감사·컴플라이언스 보고.
- 확인 필요 항목: 관련 입찰·거래 범위, 경쟁사 접촉자, 회의·메신저·메일·협회 자료, 가격 산정 자료, 자료보전 범위, leniency·신고 제도는 live source와 변호인 확인.

### 영업비밀

- 진입 신호: 전직자 자료 반출, 경쟁사 사용 의심, 소스코드·도면·고객명단 유출 주장, 경고장·고소장·가처분 관련 연락.
- 관할/신고 축: 경찰·검찰, 법원 가처분·민사절차, 내부 정보보호·인사·IT 보안 라인.
- 확인 필요 항목: 영업비밀 특정, 비밀관리 조치, 접근권한, 다운로드·반출 로그, 반출 매체, 퇴직·전보 절차, 상대방 자료 수령·사용 사실.

### 산업기술

- 진입 신호: 산업기술·국가핵심기술 유출 의심, 해외 이전·공동연구·투자·M&A 관련 자료요구, 보안감사·수사기관 연락.
- 관할/신고 축: 산업통상자원부 등 소관부처, 수사기관, 보안·수출통제·외국인투자 검토 라인.
- 확인 필요 항목: 기술 분류 후보, 이전·접근 경로, 해외 법인·협력사·투자자 접촉, 연구자료·설계자료 접근 로그, 승인·신고 필요성의 live source 확인.

### 임금체불

- 진입 신호: 고용노동부 진정·고소, 근로감독관 출석 요청, 퇴직자 체불 주장, 집단 민원, 체불확인 관련 연락.
- 관할/신고 축: 고용노동부 진정·고소 절차, 근로감독 대응, 민사·집행 절차 가능성, 내부 HR·재무 보고라인.
- 확인 필요 항목: 근로자 지위, 임금 항목, 산정 기간, 지급일, 근로시간·휴게 기록, 급여대장·계좌이체 내역, 합의·지급 커뮤니케이션.

### 무허가 영업

- 진입 신호: 감독기관 점검, 지자체·소관부처 자료요구, 영업정지·시정명령 사전통지, 경쟁사·고객 신고, 플랫폼 입점 제한 연락.
- 관할/신고 축: 소관 중앙행정기관, 지자체, 경찰·검찰, 업종별 협회·플랫폼·계약상 notice.
- 확인 필요 항목: 실제 영업 모델, 장소·온라인 채널, 허가·등록·신고 보유 현황, 신청·갱신 이력, 광고·계약 문구, 매출 발생 시점, 관할·서식·기한의 live source 확인.
